## Added

- **Named crops in the `{% raw %}{% image %}{% endraw %}` tag.** Pass a `size`
  kwarg to build the `<img>` from a named crop/resize group instead of the
  default responsive widths. The whole group becomes its own srcset — each crop
  keeps its aspect ratio — so a square thumbnail set, a wide banner set, etc.
  each get correct `srcset`/`width`/`height`:
  `{% raw %}{% image 'static/photo.jpg', size='thumb', alt='', sizes='240px' %}{% endraw %}`.
  The `size` name matches a named entry in your `images.sizes` config. Widths
  come from the compile cache, so soft crops (no fixed height) resolve too.
- **`size` on the `srcset` filter.** Same idea for the raw attribute — pass the
  group name as the second argument:
  `{% raw %}{{ 'static/photo.jpg' | srcset: 'thumb' }}{% endraw %}` →
  `static/photo-thumb-480w.webp 480w, static/photo-thumb.webp 960w`.
- **GFM footnotes** in markdown — `[^1]` references and `[^1]:` definitions
  render as linked footnotes in both template engines, via
  `marked-github-footnote`.

## Changed

- **Requires [poops-images](https://github.com/stamat/poops-images) ≥ 1.2.1.**
  Named-size widths are read from its compile cache — each named variant is now
  tagged with its size name, and the largest of a group drops the width suffix
  (`photo-thumb.webp`). The peer dependency is pinned to `>=1.2.1`.
- `marked` upgraded 9 → 18 (renderer hooks now take a token object).
- README and the images-gallery doc now reach named crops through the `size`
  kwarg instead of the old path-suffix trick.

## Fixed

- **ToC skips `.sr-only` headings.** `renderToc` no longer lists visually-hidden
  headings — a visible table of contents linking to invisible content is
  confusing.

## Footnotes in action

Named crops keep their own aspect ratio.[^1]

[^1]: That's why they're deliberately kept out of the default responsive srcset.
